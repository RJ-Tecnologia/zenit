# GEMINI.md

## Project Overview

**Zenit Finance (Web)** is a personal finance management application designed for young adults (students, recent graduates, or early-career professionals) with low-to-mid income. The core focus of the project is simplicity, visual clarity, and helping users build the habit of financial tracking without complexity.

### Main Technologies

- **Frontend:** React 19 (using the new React Compiler via `@vitejs/plugin-react` and `babel-plugin-react-compiler`).
- **Build Tool:** Vite.
- **Language:** TypeScript.
- **Routing:** [TanStack Router](https://tanstack.com/router) (file-based routing located in `src/pages`).
- **Styling:** [Tailwind CSS v4](https://tailwindcss.com/) (using the `@tailwindcss/vite` plugin).
- **Authentication:** [Better Auth](https://www.better-auth.com/) (configured in `src/lib/auth-client.ts`).
- **Validation:** [Zod](https://zod.dev/) (used for environment variable validation and schema definitions).
- **Tooling:** [Biome](https://biomejs.dev/) for high-performance linting and formatting.
- **Icons:** Lucide React.
- **UI Components:** Radix UI and Shadcn UI.

### Architecture

- **`src/pages/`**: Contains the file-based route definitions (e.g., `__root.tsx`, `index.tsx`, `login/index.tsx`).
- **`src/components/`**: Houses reusable UI components and feature-specific components.
- **`src/lib/`**: Contains shared utility libraries and client initializations (e.g., `auth-client.ts`, `utils.ts`).
- **`src/env.ts`**: Manages environment variable validation using Zod.
- **`src/route-tree.gen.ts`**: Automatically generated by TanStack Router based on the `src/pages` directory.
- **`@/*` Alias**: Configured to point to the `src/` directory for cleaner imports.

## Building and Running

The project uses `pnpm` as its package manager. Key commands include:

- **Development:** `pnpm dev`
- **Build:** `pnpm build` (Runs type-checking followed by Vite build).
- **Linting:** `pnpm lint` (Uses Biome to check for linting and formatting issues).
- **Formatting:** `pnpm format` (Uses Biome to automatically fix formatting issues).
- **Preview Build:** `pnpm preview` (Serves the production build locally).

### Environment Variables

The application requires the following environment variables (validated in `src/env.ts`):

- `VITE_BASE_URL`: Base URL for the web application.
- `VITE_API_URL`: URL for the backend API (used by Better Auth).

See `.env.example` for a template.

## Development Conventions

- **File-Based Routing:** All new routes should be added to the `src/pages` directory. The route tree will automatically update on changes when the dev server is running.
- **Styling:** Use Tailwind CSS utility classes. Custom themes and configurations follow Tailwind CSS v4 standards.
- **Formatting & Linting:** Strictly adhere to Biome's rules. Run `pnpm format` before committing changes.
- **Imports:** Use the `@/` alias for all internal imports to maintain consistency and ease of refactoring.
- **React Compiler:** The project is configured to use the React Compiler. Ensure code follows React's rules (e.g., no mutations during render) to benefit from automatic memoization.
